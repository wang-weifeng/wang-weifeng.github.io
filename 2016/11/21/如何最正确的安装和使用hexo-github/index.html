如何最正确的安装和使用hexo+github

发表于 2016-11-21 | 分类于 nodejs |

前言

网上关于hexo+github的教程太多了，多的让人可怕，但是已经那么多了为什么我还要写这篇hexo+github教程呢？
1.网上的教程有的太老，已经不适应了。
2.网上的教程好多都是一个人写（可能有错误）然后就被许多人复制，有的简直是原文复制就是没有复制作者，然后广告多的可怕，风气真差，关于这个blog互相抄的事我在Cnode社区发过一个讨论贴——-附上链接——-关于blog文章抄袭现象
3.我会把最正确的方法写下了，给需要的人最高的效率。

为什么要用hexo+github

1.我为什么要用hexo+github搭建blog

因为我本身就在学习使用nodejs，hexo是基于nodejs的静态页面，github又是代码管理仓库的好工具

2.搭建hexo+github要了解一下下面4个知识点

1.hexo
2.github
3.git
4.nodejs
关于这4个知识点可以自行去查资料了解一下（不需要很深），不过相信既然选择去搞这个hexo博客，应该对于node是有了解的

开始搭建hexo博客

1.搭建所需要的环境

1.首先要有node环境（安装了node）可以node -v看一下node的版本号
2.安装git，可以git –version看版本号
3.注册github账号

2.利用命令生成hexo

1.在node中执行，npm install -g hexo
2.查看hexo版本号，确认是否安装成功，hexo –version
3.执行hexo init
4.进入生成文件的根目录执行hexo server就可以看到一个blog最原始的页面（blog第一步完成了）
这里只是一个本地的hexo，只能本地访问，下面来设置可以外网访问

打开注册好的github

1.选择New repository，新建一个项目。

2.填写Repository name（注意只能填写标准格式，也就是你的(注册名.github.io)就拿我的来说，就是旁边的Owner.github.io（wang-weifeng.github.io））填写完成点击Create repository

3.点击setting设置按钮添加ssh（这一步在你安装github时应该也会设置的不详细介绍）

4.点击具有ssh这一项的设置按钮，打开会看到添加ssh选项(自行查找如何设置)

5.我觉得最后一个关于github的设置很关键，也是在许多教程中没有说的，也是导致我为什么hexo无法上传到github的原因
5.1打开刚开始创建的项目，在这个项目中找到settings选项，如图

5.2往下翻找到github pages，然后会看到这一栏最下面有个选择是打开github pages，因为我打开过了显示的是Launch automatic page generator,根据步骤一步一步，直到你在浏览器中打开你的域名（之前设置的项目名称）比如我的就是wang-weifeng.github.io有成功内容

直到这里github已设置完毕

将hexo生成的文件生成静态文件

1.因为有的会出现问题所以每次可以执行
hexo clean
2.hexo g
3.hexo d
然后你在浏览器中打开你的域名（之前设置的项目名称）比如我的就是wang-weifeng.github.io
